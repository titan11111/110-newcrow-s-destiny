# 各ボスの動き・攻撃方法一覧（CROW'S DESTINY）

設計書「浄化の儀式ボスデザイン完全設計書」に基づき、各面ボスは専用の攻撃パターンを実装しています。  
HP30%以下でピンチフェーズ（動き・弾が強化、新技が出現）に移行します。

## 攻撃フェーズ詳細（全ボス共通ロジック）

| フェーズ | 名前 | 内容 | ステージによる変化の要点 |
|----------|------|------|---------------------------|
| **Phase 0** | 全方位弾 | 中心から等間隔で弾を放射。角度が時間で回転。 | 弾数 6+idx×2。idx≥4 で2周目（位相ずれ）追加。発射間隔は idx が大きいほど短い。 |
| **Phase 1** | 突進バースト | プレイヤー位置へ突進→到達で全方位に弾拡散。 | 弾数 10+idx×3、突進速度 7+idx×0.6。idx≥5 で2重バースト（遅延弾）。 |
| **Phase 2** | 狙い弾＋召喚 | プレイヤー方向へ直線弾（扇状に1〜4本）。一定間隔で雑魚召喚。 | 狙い弾本数 1+min(idx,3)。idx≥6 で雑魚2体同時。召喚間隔は idx で短縮。 |
| **Phase 3** | スパイラル弾 | 回転する弾を連続発射。 | idx≥2 で2重スパイラル。idx≥6 で各スパイラルに追加弾。発射間隔は idx で短縮。 |
| **Phase 4** | レーザー斉射 | 予告線表示→プレイヤー方向へ赤いビーム状弾を複数本一斉発射。 | ステージ3（idx≥2）以降のみ出現。本数 7+idx×2、スプレッド・速度は idx で増加。 |

※ フェーズの長さは `phaseDur = max(160, 280 - idx*18)` なので、ステージが上がるほど**フェーズが短くなり、パターン切り替えが速い**です。

## 難易度を徐々に上げるための案

現状は「ステージ番号（idx）」に比例して弾数・速度・フェーズ数が増えているので、その延長と追加案をまとめます。

### 案A：既存パラメータの段階的強化（実装しやすい）

- **体力カーブ**  
  - 現状: stages の `bossHpBase` で 220→660。  
  - 案: 後半だけもう一段上げる（例: 5〜7章で +40〜80）、または「章ごとに前章比 +8%」などにすると、後半の持久戦感が出る。
- **フェーズ時間の短縮**  
  - 現状: `phaseDur = max(160, 280 - idx*18)` で 7章は約172フレーム。  
  - 案: 5章以降で `phaseDur` をさらに -10〜15 すると、同じ時間で多くのパターンを見せることになり難易度アップ。
- **攻撃速度（atkSpd）**  
  - 現状: `bossAtkSpd` が 1.0→1.7。  
  - 案: 6・7章で 1.8〜2.0 まで上げると、弾幕の「間」が詰まる。
- **弾速・弾数**  
  - 現状: `bulletMul = 1 + idx*0.15`、各フェーズで idx に応じて弾数・速度が増加。  
  - 案: 5章以降で `bulletMul` にさらに +0.1 や「idx≥5 のとき弾数+2」などを入れると、後半だけきつくできる。

これらは **stages.js の数値**と **boss.js の phaseDur / bulletMul / intv など**の微調整で実現できます。

---

### 案B：HP帯で行動変化（中〜高難度）

- **残りHPでフェーズが増える / 攻撃が激化する**  
  - 例: 残り 50% 以下で「Phase 4 のレーザー本数 +3」「Phase 0 の2周目を全ボスで有効」など。  
  - 実装: `boss.js` の各フェーズ内で `if (this.hp / this.maxHp < 0.5)` のときだけ弾数・間隔を追加で変更する。
- **残り 25% で「ラストスパート」**  
  - 例: フェーズ時間を一気に短くする（phaseDur を一時的に 0.7 倍など）、またはレーザー予告時間を短くする。  
  - プレイヤーに「もう一踏ん張り」の緊張感を与えつつ、難易度のピークを後半に持っていける。

---

### 案C：ボス固有の「ワンアクション」追加（差別化）

- 各ボスに**1つだけ**固有の動き or 攻撃を足す。  
  - 例:  
    - ステージ2: 突進前に一瞬「溜め」ポーズを入れ、突進速度を少し上げる。  
    - ステージ4: Phase 2 の狙い弾の前に、自分が短距離ダッシュで位置をずらす。  
    - ステージ6: Phase 4 のレーザーを「2連続で角度を少しずらして発射」する。  
  - 実装: `boss.js` の `update` 内で `this.idx === 1` などで分岐し、既存フェーズの前後にちょとした移動や遅延を入れる。
- こうすると「各ボスの動き・攻撃一覧」に「固有アクション」の列を1行足して説明しやすくなり、難易度というより**印象と攻略の違い**をはっきりさせられる。

---

### 案D：弾の「質」の変化（上級者向け）

- **遅延弾・2段弾**  
  - 例: Phase 0 の弾が、一定時間後に速度アップ or 分裂する。  
  - 現状の「2周目」を「少し遅れて出る2段目」にすると、避け方の難易度が上がる。
- **屈曲弾**  
  - 例: Phase 2 の狙い弾のうち1本を「途中で角度が変わる」ようにする（目標位置を途中で更新する等）。  
  - 実装は弾オブジェクトに `phase` や `t` を持たせ、一定フレーム後に向きを更新する形が扱いやすい。
- **通過弾（貫通）**  
  - 例: Phase 4 のレーザーの中心1本だけ「画面端まで消えない」長い当たり判定にする。  
  - 避け方の選択肢が増え、難易度と見た目のインパクトを同時に上げられる。

---

### 実装の優先度の目安

1. **まず案A**  
   - 既存の `idx` ベースの難易度を少し強めるだけで、バランス調整として安全。  
2. **次に案B（HP帯）**  
   - 同じボスでも後半が忙しくなり、難易度の「段階的上昇」がはっきりする。  
3. **余裕があれば案C（固有アクション）**  
   - 一覧表の「動き・攻撃」に個性を足せる。  
4. **案D**  
   - 新規弾ロジックが必要なので、仕様を決めてから追加するのがおすすめ。

必要なら、案Aの具体的な数値例（例: 5章以降の `phaseDur` の式、`bossHpBase` の表）をコードレベルで書き起こすこともできます。
