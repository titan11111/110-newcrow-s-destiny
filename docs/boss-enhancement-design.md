# ボス演出・動き強化 設計書（CROW'S DESTINY）

参照: ボス強化案アドバイス。世界観を深めつつ、より面白くプレイできるよう各ボスの演出・動きを強化する設計です。

**関連ファイル**: `js/entities/boss.js`（各 `updateBoss*`）, `js/render/fx.js`（burst, addFloorCrack, arenaFreeze 等）, `js/game.js`（state, boss 生成）

## 1面 ／ 穢れの先兵 — 彷徨う巨骸

**実装箇所**: `boss.js` の `updateBoss1`、コンストラクタに `turnFlashTriggered` 等のフラグ追加可。`fx.addFloorCrack`, `fx.shake` は既存。

### ① イージング付き往復＋折り返し衝撃

- **目的**: 巨骸らしい「重力を感じる」重厚な左右往復。端で減速→一瞬停止→折り返し。
- **現状**: `this.x += this.moveDir * lrSpeed` の等速往復。`didTurn` 時に `fx.shake` / `addFloorCrack` は既に発火。
- **設計**:
  - 移動を **正弦波の正規化値 + イージング** に変更。例: `normalizedX = sin(t * 0.015)` で -1〜1 を生成し、`easedX = sign(normalizedX) * pow(abs(normalizedX), 0.6)` で端付近をゆっくりに。
  - `boss.x = centerX + easedX * moveRange` で位置更新。`moveDir` は `easedX` の符号から算出。
  - **折り返し検出**: `|normalizedX| > 0.97` のとき「端到達」とみなし、1回だけ `turnFlashTriggered = true` で演出発火。`fx.shake = 35`、`fx.addFloorCrack(boss.x, H-5, 55)`、SE。`|normalizedX| < 0.9` で `turnFlashTriggered = false` にリセット。
- **変数案**: `boss1WaveT`（タイマー加算）、`turnFlashTriggered`（boolean）。

### ② 紫炎噴射の前兆を3段階化

- **目的**: 2秒前〜発動まで「口元の火花→チャージ膨らみ→全体紫点滅」で緊張を段階的に上げる。
- **現状**: `purpleBeamTelegraph` が 120 フレーム（2秒）で 0 になると発動。
- **設計**:
  - 120〜81: **段階1** — 口元・胸骨に紫の小火花パーティクルを `fx.burst` で少量（2〜3個）ずつ追加。または draw で目/口周りに小さな円を描画。
  - 80〜41: **段階2** — ボス中心に紫のチャージ円を描画し、`telegraph/80` で半径が 0→30 程度に拡大。
  - 40〜1: **段階3** — ボス全体を紫で multiply または fill で点滅（`Math.floor(telegraph/8)%2` で ON/OFF）。
  - 描画は `boss.js` の `draw` 内で `idx === 0 && purpleBeamTelegraph > 0` の分岐を拡張するか、`game.js` の draw でボス描画後にオーバーレイを描く。

### ③ HP段階的崩壊ビジュアル

- **目的**: HP50%以下で骨の欠け、30%以下で紫炎漏れ、10%以下（断末魔）で骨剥がれデブリ。
- **設計**:
  - **50%以下**: スプライトの一部をマスクするか、別コマ（HIT や専用「欠損」コマ）に切り替え。または draw で `globalCompositeOperation = 'destination-out'` で矩形を抜く等で「欠けた」表現。
  - **30%以下**: 毎フレーム `fx.burst(boss.x, boss.y, purple, 2, 1)` で紫の小パーティクルを常時追加。または `fx.p` に寿命の長いパーティクルを追加して体から漏れる炎を表現。
  - **10%以下（lastStandFreezeT 中）**: 断末魔停止中、`fx.addArenaDebris` を多めに呼び、骨片が剥がれ落ちるように。既存の `lastStandTriggered` / `lastStandFreezeT` と連携。

## 2面 ／ 粘体の母胎 — ハイヴコア

**実装箇所**: `updateBoss2`、コンストラクタの `boss2Phase === 'intro'` 時の座標・登場処理。`fx` に粘液用パーティクルや軌跡用の配列が必要なら `fx.js` 拡張。

### ① 上方降下登場＋粘液滴下

- **目的**: 画面上方から「滴り落ちる粘体」として降下。着地時に粘液飛散。
- **現状**: コンストラクタで `this.x = CFG.W/2; this.y = -120`、`arrived` まで `this.y` を補間で下げている（`update` 内の `!arrived` 分岐）。
- **設計**:
  - 登場中（`!arrived`）の間、ボス真下に **緑の粘液パーティクル** を毎フレーム 1〜2 個追加。`vy` 正で下に落ちる。`fx.burst(this.x, this.y, '#00ffaa', 2, 0)` で下方向に弱い初速を付ける等。
  - `arrived` になった瞬間に `fx.burst(this.x, this.y, '#00ffaa', 30, 5)` で着地飛散。必要なら `fx.addFloorCrack` を短めの duration で呼び「床に付いた」表現。

### ② 暴走中のジッター＋残像最大化

- **目的**: 暴走＝「制御を失った」動き。小刻み振動＋軌跡の残像を常時出す。
- **現状**: `isEnraged` 時に `moveRangeX`/`moveSpeed` が増加。残像は未実装の可能性。
- **設計**:
  - 暴走中は `this.x += rr(-3, 3)`、`this.y += rr(-3, 3)` でジッター（clamp で範囲内に収める）。
  - 軌跡は `energyTrail` に相当するものをボス2用に追加。`update` で `this.boss2Trail = this.boss2Trail || []` を push し、最大 30 本等にして draw で線または円を描画。暴走中は trail の寿命を長くして「常時最大本数」に近づける。
  - 暴走開始時、`boss2RageFlashT = 180` のようなタイマーを設け、`Math.floor(boss2RageFlashT/4)%2 === 0` の間は描画で tint や alpha を切り替えて断続点滅。

### ③ 子機召喚を「産卵」演出に

- **目的**: 母胎らしく、膨張→ピークで子機出現→収縮。
- **現状**: `enemies.push(new Enemy(...))` でサテライトを追加している箇所を検索。ボス2の `phase === 2` で敵を召喚している部分。
- **設計**:
  - 召喚前に **膨張フェーズ** を入れる。`boss2SpawnScale = 1` → 30 フレームで 1.15 → 15 フレームで 1.0 に戻す。この間は `update` で `this.boss2SpawnScale` を更新し、`draw` で `c.scale(boss2SpawnScale, boss2SpawnScale)` を適用。
  - スケールがピーク（1.15）のフレームで `enemies.push(...)` を実行。その後しぼむアニメ（1.0→0.92→1.0 を 20 フレーム等）を入れる。
  - ピンチ時の 4 体召喚では膨張を 1.25 まで大きくし、収縮もやや長めにすると「産んで力を使い果たした」印象に。

## 3面 ／ 擬態する知性 — ミミック

**実装箇所**: `updateBossMimic` のテレポート先決定、フェイク時の残像描画、`mirrorClones` の見た目。

### ① テレポート先をプレイヤー周囲に偏らせる

- **目的**: ランダムではなく「取り囲む」意図を感じさせる。
- **現状**: `this.x = rr(80, W-80); this.y = rr(70, H*0.6)` の完全ランダム。
- **設計**:
  - 70% で「プレイヤーを中心とした特定角度」に出現。例: プレイヤーから見て左・右・左斜め上・右斜め下 など、角度を 4〜6 パターンに限定し `px + cos(angle)*200` のような形で目標座標を算出。30% は従来どおりランダム。
  - `portalResidue` は既存のまま、出現位置に残像を描けばよい。

### ② フェイクテレポートの見分けやすさ

- **目的**: フェイクは残像の色を薄く、消え方を「スッと消える」にして上級者が見抜けるように。
- **現状**: 20% でフェイク（実際はテレポートしない）。残像は `afterimages` に追加されている。
- **設計**:
  - フェイク時は `afterimages` に渡すオブジェクトに `isFake: true` を付与。draw で `isFake` なら `globalAlpha *= 0.6`、色を白っぽく（`#ddd` 等）。
  - フェイクの消滅時（残像の t が 0 に近い）は、本物の残像のようにパーティクルで散るのではなく、`globalAlpha` を線形で 0 にするだけの「スッと消える」描画にする。

### ③ ミラーダブルの精度をHP連動で劣化

- **目的**: HP 低下で擬態が崩れ、コピーが不安定に見える。
- **設計**:
  - HP60%以上: 現状どおりコピーを同じ色で描画。
  - HP40〜60%: コピーに `globalAlpha = 0.85` や `fillStyle` を少し薄い紫に。
  - HP30%以下: コピーを `Math.floor(timer/4)%2 === 0` で点滅。または `timer` で揺らすオフセットを加えて「激しく不安定」に見せる。

## 4面 ／ 蒼穹の守護者 — 鉄翼

**実装箇所**: `updateBossIronWing`、`_ironWingNextPhase`。draw でバンク角・排気炎。デスロールは既存の `deathRollT` を参照（現状は未使用の可能性あり）。

### ① PATROL中のバンク傾き＋排気炎

- **目的**: 飛行機らしい「速度変化＋進行方向へ傾き＋後方に炎」。
- **設計**:
  - **速度イージング**: PATROL の `ironWingVx` を、目標速度に対して `lerp(current, target, 0.1)` で滑らかに。または sin で「加速→定速→減速」のサイクルを 1 往復分で表現。
  - **バンク角**: `ironWingBankAngle`（度）を `ironWingVx` の符号と大きさで決定。例: `vx < 0` なら -10°、`vx > 0` なら 10°。draw で `c.translate(boss.x, boss.y); c.rotate(bankAngle * Math.PI/180);` してからスプライト描画。
  - **排気炎**: PATROL 中のみ、毎フレーム `fx.burst(this.x - 30, this.y, '#ff4400', 2, 1)` で後方（左向きなら x マイナス側）に小さい赤パーティクル。または `fx` に `addTrailParticle(x, y, color, life)` を追加し、軌跡を短い線で描く。

### ② ダイブ前の助走（画面上部で停止＋発光）

- **目的**: 急降下の 0.5 秒前に画面上部に引き寄せられ、一瞬停止してから発光→急降下。
- **現状**: `phase === 'dive'` で `ironWingDiveTargetX/Y` に向かって直進。
- **設計**:
  - ダイブフェーズに入ったら **サブフェーズ** を設ける。`diveSubPhase = 'approach_top'`（画面上部へ移動）→ `'hold'`（0.3秒停止）→ `'dive'`（現行の突進）。
  - `approach_top`: 目標を `(W/2, 60)` などにし、`lerp` で上に引き寄せる。到達したら `hold`、`hold` は 18 フレームで `dive` へ。
  - `hold` 中はボスを赤く multiply 描画し、排気炎のパーティクルを多めに出す。`dive` 突入時に `fx.shake = 22` は既存でよい。

### ③ デスロール軌跡を残留炎床化

- **目的**: 通過した軌跡に「残り時間付きの炎床」を残し、逃げ場がなくなるプレッシャーを視覚化。
- **設計**:
  - `deathRollPath` または新規 `ironWingDeathRollTrail` に `{ x, y, t }` を 5 フレームごとに push。`t` は 60 で初期化し、毎フレーム `t--`。0 以下で削除。
  - draw で配列を走査し、各要素を赤い円（半径 20〜30）で描画。`globalAlpha = t/60` でフェードアウト。当たり判定は既存の `deathRollT` と軌跡の円で別途判定するか、現行どおり「ボス本体のみ」でも可（演出優先なら軌跡は視覚のみ）。

## 5面 ／ 門番 — 多脚のガーディアン

**実装箇所**: `updateBossGuardian`。`domeShieldT`、`overdriveT`、`legStabT` 等。

### ① 縄張り意識（プレイヤーが下に行くと前進して押し戻す）

- **設計**:
  - プレイヤー `py` が `H * 0.6` より下なら、ボスの目標 Y を `ty + 80` のように下にシフト。`py` が上なら通常の `ty`。`this.ty` を毎フレーム `lerp(this.ty, targetTy, 0.03)` で更新し、ボスが「前進して圧迫→引き下がる」ように見せる。
  - 目標 X は現状の正弦波のままでもよい。

### ② オーバードライブ起動シーケンス（10秒）

- **設計**:
  - `overdriveT` を「発動までのカウントダウン」と「発動中の残り時間」の 2 段階に。例: `overdriveWarnT = 600`（10秒）で開始。0 になるまでボスは攻撃を弱めるか止め、体を白く発光させる。`overdriveWarnT === 1` で `overdriveT = 900` にし、本発動。
  - 警告中は `txt.show('OVERDRIVE INITIATING', '#fff', 60, 24, W/2, 80)` のような表示。armorPeelLevel の見た目（装甲がめくれる）は draw でスプライトにひび割れ線を描くか、別レイヤーで上から描画。

### ③ レッグスタブの予備動作（のけぞり→踏み込み）

- **設計**:
  - `legStabT` の最初の 12 フレームを「予備」とする。この間は `this.y -= 2` のように上にのけぞる（または draw でオフセット Y を -20 にする）。その後、脚の弾を出す処理は既存どおり。
  - 脚が画面に到達するタイミング（既存の `legStabT % 4 === 0` で弾を出している付近）で `fx.addFloorCrack(脚のX, legY, 40)` と `fx.shake = 15` を追加。

## 6面 ／ 蒼穢の女王 — ブルーコア

**実装箇所**: `updateBossBluecore`。`snowQueenGuard`、`snowQueenEnraged`、`blizzardT`。`fx.arenaFreeze` は既存。

### ① 氷盾のチャージ儀式（1.5秒・中央で静止）

- **設計**:
  - 盾を張る行動を選択したら、即座に SHIELD にするのではなく **チャージフェーズ** を挟む。`snowQueenGuardChargeT = 90`（1.5秒）をセットし、この間は `snowQueenMoveTargetX/Y` を画面中央に固定し、攻撃も行わない。draw でボス周囲に氷の結晶（小さい円や線）がゆっくり集まるアニメ。
  - 90 フレーム後に `snowQueenGuard = true`、`snowQueenGuardT = 150`。プレイヤーは「チャージ中に攻めろ」と読める。

### ② Enrage突入の停止→膨張→暴走

- **設計**:
  - `snowQueenEnraged` が true になった瞬間、`snowQueenEnrageHoldT = 30` をセット。この間は移動・行動を止め、draw でスケール 1.0→1.2→1.0 を 30 フレームでアニメ。同時に `fx.arenaFreeze = 60` を発動させて画面端を凍らせる。
  - 30 フレーム後に通常の Enrage 行動（鋭角移動等）を再開。

### ③ ブリザード前兆（画面に白ノイズが増える）

- **設計**:
  - ブリザード発動の 30 フレーム前から `blizzardWarnT = 30` をカウントダウン。この間、draw で画面全体に `fillStyle = 'rgba(255,255,255, 0.03 * (30 - blizzardWarnT))'` の矩形をランダムな位置に小さく多数描く（ノイズ風）。または canvas の `getImageData` でノイズを乗せる。
  - 発動瞬間に `fx.flash` を白で一瞬光らせてから、既存のブリザード弾幕を開始。

## 7面 ／ 裂け目そのもの — ヴォイド

**実装箇所**: `updateBossVoid`。形態移行、`voidAfterimages`。BGM 切りのタイミングは `game.js`。

### ① 形態移行を画面全体の崩壊で表現

- **設計**:
  - 形態 0→1: 移行開始時に `fx.arenaVoidNoiseT = 90` のような新エフェクトを追加。draw で紫〜マゼンタの横線をランダムに 20 本ほど描画し、alpha を時間でフェード。ボスは 30 フレーム非表示にしてから、スケールを 0.5→1.2 で再表示。
  - 形態 1→2: BGM を一瞬 mute（`sound.stopBGM()` で 0.5 秒待つか、game.js で state を「暗転」に）。`fx` に `fullScreenDarkT = 30` をセットし、draw で画面を `fillStyle = '#000'` で真っ暗に。その後ボスを別位置に再配置して出現。

### ② 収束弾幕＋四隅から霧が侵食

- **設計**:
  - 弾に `glow: true` や trail 用の配列を持たせ、draw で弾の後方に細い線や薄い円を描く。
  - ヴォイドアポカリプス発動時、`fx.voidCornerFogT = 180` をセット。draw で四隅から中央に向かって `fillStyle = 'rgba(128,0,255,0.1)'` の矩形を段階的に広げる。プレイヤーが弾幕を抜けたら（または時間経過で）`voidCornerFogT` を 0 にして霧を晴らす。

### ③ 残像からの遅延追尾弾（残滓攻撃）

- **設計**:
  - `voidAfterimages` の各要素に `t`（経過フレーム）を持たせる。既存の `VOID_AFTERIMAGE_LIFE` に加え、`t === 60`（2秒後）のときにその残像の座標から追尾弾を 1 発 `bullets.push`。弾オブジェクトに `homing: true` と `color: '#cc44ff'` を付与。
  - 残像は最大 5 本なので、5 体それぞれから 2 秒後に 1 発ずつ出ると、時間差で追尾弾が飛んでくる「残滓攻撃」になる。

## 実装時の共通メモ

- **fx.js の拡張**: 粘液軌跡、炎床の配列、四隅の霧、ノイズ用タイマーなど、必要に応じて `FX` クラスにプロパティと `update`/`drawArenaEffects` 内の描画を追加する。
- **game.js**: ボス描画の前後にオーバーレイ（紫炎前兆の円、氷の結晶など）を描く場合は、`game.js` の draw ループで `boss` の状態に応じて分岐する。
- **音**: 折り返し衝撃、オーバードライブ警告、形態移行の「無音」など、既存の `opts.sound` や `sound.playBossCharge()` 等を流用し、不足分は `sound.js` にメソッド追加を検討。
