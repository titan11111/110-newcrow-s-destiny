# ステージ6バグ修正（白いE・画面埋まり）

## 現象

- ステージ6で画面が白い「E」のようなオブジェクトで埋まり、カクつく・見づらい
- 敵や弾が異常に多く表示される

## 想定原因

1. **スプライト画像未読み込み・破損**  
   敵（enemy6 / enemy7）の画像がロード失敗や 0 サイズのとき、スプライト描画パスで `drawImage` が不正になり、ブラウザのプレースホルダーや崩れた表示（E 状など）が出る可能性。
2. **敵・弾のスポーン過多**  
   スポーン間隔や削除タイミングの関係で、同時に存在する敵・弾が増えすぎて画面が埋まる。

## 実施した修正

### 1. 敵描画のフォールバック（`js/entities/enemy/draw.js`）

- `e.spriteKey` があり `IMG[e.spriteKey]` が存在する場合でも、  
  **`naturalWidth` または `naturalHeight` が 0 のときはスプライト描画を行わない**ようにした。
- その場合は従来のフォールバック（楕円＋目などの汎用敵）で描画する。
- 画像未読み込み・破損時にも「E」状の表示にならず、確実にフォールバックされる。

### 2. 敵の同時出現数上限（`js/game/spawn.js`）

- `MAX_ENEMIES_ON_SCREEN = 28` を定義。
- `spawnEnemies` 内で `game.enemies.length >= MAX_ENEMIES_ON_SCREEN` のときは新規スポーンをしない。
- ステージ6に限らず、全ステージで敵が画面を埋め尽くすことを防ぐ。

### 3. 敵弾の同時存在数上限（`js/core/objectPool.js`）

- `MAX_ACTIVE_BULLETS = 220` を定義。
- 敵弾用アダプタの `push` で、既に active が 220 以上なら新規追加しない（`return null`）。
- 弾が際限なく増えて画面が埋まるのを防ぐ。

## 確認してほしいこと

- ステージ6で白い「E」が大量に出ないか
- 敵・弾が多すぎて画面が埋まらないか
- 通常プレイで「敵が出なさすぎる」「弾が消えすぎる」などの違和感がないか

問題が続く場合は、`images/enemy6.png` および `images/enemy7.png` の存在・読み込み可否を確認してください。
