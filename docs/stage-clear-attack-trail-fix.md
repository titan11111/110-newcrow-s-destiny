# ステージクリア後のオート攻撃・残像が残る問題 — 改善案3パターン

## 原因

1. **STATE.STAGE_CLEAR 中も `crow.update(keys, d)` が呼ばれている**  
   → カラスのオート攻撃（`shoot()`）が止まらず、弾（feathers）が追加され続ける。

2. **`startStage()` でプレイヤー弾・ダッシュ残像をクリアしていない**  
   → `crow.feathers` と `crow.dashTrail` がそのまま次のステージに持ち越される。

3. **ボス撃破直後も攻撃・残像の「止める」処理がない**  
   → 撃破フレームから STAGE_CLEAR 表示〜次ステージ開始まで、弾と残像が描画され続ける。

---

## 改善案 3パターン

### パターン1: クリアのみ（最小変更・確実）

**内容**

- ボス撃破で **STATE.STAGE_CLEAR に移行した直後** に、プレイヤー弾・ダッシュ残像をクリアする。
- **STAGE_CLEAR 中は `crow.update` で射撃しない**（`shoot()` を呼ばない）。
- **`startStage()` 内でも** `crow.feathers` と `crow.dashTrail` を空にして、次ステージに絶対に残らないようにする。

**メリット**  
実装が簡単で、確実に「次のステージでは弾も残像も出ていない」状態にできる。

**デメリット**  
撃破直後、画面上の弾・残像が「パッ」と消える（演出はなし）。

---

### パターン2: 撃破直後に「攻撃停止＋残像だけフェード」

**内容**

- ボスが倒れた瞬間に「攻撃禁止」フラグを立てる（例: `game.stageClearCombatLock = true`）。
- **STAGE_CLEAR 中**は `crow.update` で **射撃処理をスキップ**（移動やアニメは続けてもよい）。
- 既存の **feathers / dashTrail は「更新はするが新規追加なし」** にして、  
  時間経過で画面外に出る or 寿命で消えるまで描画する（自然に消える）。
- **`startStage()` で feathers / dashTrail をクリア**し、次ステージでは絶対に残らないようにする。

**メリット**  
撃破後も「いま出ている弾・残像」は自然に消えていくので、やや自然な見た目になる。

**デメリット**  
STAGE_CLEAR 時間が短いと、まだ弾や残像が残ったまま次ステージに切り替わる可能性があるので、`startStage()` でのクリアは必須。

---

### パターン3: 演出付き（撃破→一瞬止める→クリア）

**内容**

- ボス撃破直後、**短時間（例: 0.3〜0.5秒）だけ「戦闘フリーズ」**  
  → この間は `crow.update` で射撃も移動も行わない（または移動のみにして射撃は止める）。
- その後 **STATE.STAGE_CLEAR に移行**し、そのタイミングで **feathers / dashTrail を一括クリア**。
- 必要なら「クリア瞬間に残像を一瞬フェードアウトさせる」演出（alpha を一気に 0 にする）を追加。

**メリット**  
「止まった→弾・残像が消えた→クリア表示」の流れがはっきりし、見た目が一番整う。

**デメリット**  
実装量がやや増える（フリーズ時間の管理、状態分岐の追加）。

---

## 推奨

- **手軽に直したい** → **パターン1**（STAGE_CLEAR 移行直後と startStage() の両方でクリアし、STAGE_CLEAR 中は射撃しない）。
- **見た目を少しだけ自然にしたい** → **パターン2**（攻撃停止＋既存弾・残像は自然消滅＋startStage でクリア）。
- **演出まで込みで整えたい** → **パターン3**（短い戦闘フリーズ＋クリア時に一括クリア／フェード）。

---

## 実装状況

**パターン3を採用済み**（撃破→0.5秒フリーズ→弾・残像クリア→STAGE_CLEAR）

- `js/game/constants.js`: `STATE.STAGE_CLEAR_FREEZE` を追加、`PAUSABLE_STATES` に含めた。
- `js/game.js`: ボス撃破時に `STATE.STAGE_CLEAR` ではなく `STATE.STAGE_CLEAR_FREEZE` に遷移。`STAGE_CLEAR_FREEZE_DUR`（30フレーム≒0.5秒）経過後に `crow.feathers` / `crow.dashTrail` をクリアしてから `STATE.STAGE_CLEAR` へ移行し、「STAGE CLEAR」テキストを表示。`startStage()` 内でも `feathers` / `dashTrail` をクリアして次ステージに残らないようにした。
